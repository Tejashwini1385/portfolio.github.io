10. Walmart Live Inventory

Description

Walmart‚Äôs live inventory system uses hash tables to map unique SKUs (product IDs) to inventory data like stock, location, and pricing. This enables fast, real-time updates and lookups across stores.

SKUs are hashed into buckets, and collisions (when multiple SKUs map to the same bucket) are resolved using chaining (linked lists).

Hash tables allow constant-time operations on average, but in large-scale systems like Walmart‚Äôs, a few key challenges arise:

  • Handling hash collisions efficiently to avoid lookup delays.
  • Managing concurrent updates across distributed servers.
  • Keeping data consistent across caches and databases.
  • Scaling the hash structure to handle millions of products.

HashMap Insert Operation in Java

class HashMapExample {
    static final int SIZE = 10; // size of hash table
    static class Node {
        String key;
        String value;
        Node next;
        Node(String key, String value) {
            this.key = key; this.value = value; this.next = null;
        }
    }
    
    Node[] table = new Node[SIZE];
    
    int hash(String key) {
        return Math.abs(key.hashCode()) % SIZE;
    }
    
    void put(String key, String value) {
        int index = hash(key);
        Node head = table[index];
        Node newNode = new Node(key, value);
        
        if (head == null) {
            table[index] = newNode;
            return;
        }
        
        // If key exists, update
        Node curr = head;
        while (curr != null) {
            if (curr.key.equals(key)) {
                curr.value = value;
                return;
            }
            curr = curr.next;
        }
        
        // Otherwise, insert at head
        newNode.next = head;
        table[index] = newNode;
    }
}
